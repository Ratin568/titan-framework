Titan Framework

A modular, event-driven Discord bot framework with priority-based module loading
🐍 Python 3.11+ 📦 Modular ⚡ Event-Driven 🛡️ AntiSpam Built-in 📄 MIT License
📥 Clone Repository 📚 Documentation

✨ Features

🔌

Truly Modular

Every feature is a module. Add or remove without touching core.

⚡

Priority-Based Loading

Control load order via priority in register.json.

🎯

Event-Driven

Built-in EventBus for loose coupling between modules.

🧩

Service Container

DI-style service sharing between modules.

🛡️

Production AntiSpam

Warning levels with decay timers, mute role, DMs, logging.

🚀

No Restrictions

Modules can do ANYTHING. No hand-holding, no artificial limits.

🚀 Quick Start

# Clone & Install
git clone https://github.com/YOUR_USERNAME/titan-framework.git
cd titan-framework
pip install -r requirements.txt

# Configure
cp .env.example .env
# Edit .env with your Discord bot token

# Run
python main.py

📦 Writing Your First Module

# Step 1: Create folder structure
modules/my_first_module/
├── register.json
└── main.py

# Step 2: Create register.json
{
  "name": "my_first_module",
  "priority": 50,
  "enabled": true,
  "files": ["main.py"]
}
# Step 3: Create main.py
async def setup_module(bot, event_bus, services):
    print("✅ My first module loaded!")
    
    @bot.tree.command(name="hello", description="Say hello")
    async def hello(interaction):
        await interaction.response.send_message(f"Hello from my module!")

🔧 Core Architecture

EventBus

Just a pipe – receives and sends events. No processing, no queue logic. Both sync and async subscribers.

ServiceContainer

Simple DI container. Singleton and prototype support. Factory methods.

PriorityRegistry

Recursive module discovery. Priority-based loading. Runtime enable/disable. Module reload without restart.
